# 🗺️ Smart Orchestrator Roadmap

Welcome to the future of AI-assisted Vibe Coding. This roadmap outlines the strategic direction and upcoming features for the **Smart Orchestrator** pattern and Standard Library.

## 📌 Phase 1: Foundation (Current)

- [x] Auto-Mechanism protocol definition (`AGENTS-template.md`).
- [x] Core architecture diagrams (Mermaid.js).
- [x] GitHub Community Health files (Code of Conduct, Contributing, Security).
- [x] Baseline Standard Library (Frontend, Backend, DevOps, Security, Data Science, QA, Mobile).
- [x] Basic NPX CLI tool setup.

## 🚀 Phase 2: Community & Ecosystem Expansion (Q3 2026)

- [ ] **Community Skills Directory:** A dedicated web interface or searchable registry for user-submitted `SKILL.md` templates.
- [ ] **Advanced CLI Features:** Interactive prompt to allow users to select multiple skills during initialization.
- [ ] **IDE Extensions:** Native plugins for VS Code and Cursor to instantly sync the `.skills/` folder with the latest community standards.
- [ ] **Web3 & Game Dev Skills:** Expanding the standard library to cover Smart Contracts (Solidity), Unity, and Unreal Engine best practices.

## 🧠 Phase 3: Autonomous AI Validation (Q4 2026)

- [ ] **Post-Execution Linting:** A specialized skill that allows the AI to automatically run linters and compilers to verify its own work before presenting it to the user.
- [ ] **Multi-Agent Workflows:** Templates for coordinating multiple specialized AI sub-agents (e.g., separating the "Planner" agent from the "Coder" agent).
